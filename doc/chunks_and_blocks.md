# Управление чанками и блоками в мире

В этом документе описано, как в CubeWorld устроено хранение и появление чанков и блоков: когда чанк считается «полным», что такое «недозаполненный» чанк и как мир ведёт себя по памяти.

## 1. Чанк в памяти всегда «целый»

- **Чанк** — это один объект `TChunk` с массивом **ровно 16×16×16** блоков (`FBlocks`).
- В памяти нет «половинчатого» или «недозаполненного» чанка в смысле размера: чанк либо есть, либо его нет. Если чанк есть — у него всегда 16³ ячеек, в каждой лежит `TBlock` (в том числе с `Id = 0` — воздух).

То есть:

- **«Чанк полностью заполнен»** — все 16³ ячеек содержат какой‑то блок (часть может быть воздухом). С точки зрения структуры данных ничего особенного: тот же массив.
- **«Чанк с блоками» или «недозаполненный»** (много воздуха) — то же самое: один массив 16×16×16, в части ячеек `Id = 0`.

## 2. Когда чанк появляется в мире

Чанк **создаётся и добавляется в мир только при первом обращении**:

- вызов **`SetBlock(позиция в этом чанке)`** — мир создаёт чанк, если его ещё нет;
- вызов **`EnsureChunk(координаты чанка)`** — явно «подгрузить» чанк.

Пока вы не обратились к чанку — его **нет в `TWorld`** (нет в списке `FChunks`). Тогда:

- **`GetBlock(позиция в этом чанке)`** не создаёт чанк, а возвращает «воздух» (`Id = 0`).
- **`SetBlock(...)`** создаёт чанк (очистка воздухом → загрузка из хранилища или генерация), затем записывает блок.

Итого: мир **разрежен по чанкам** (хранятся только те чанки, к которым уже обращались), но **каждый хранимый чанк всегда полный** (16×16×16 блоков).

## 3. Схема хранения

```
Мир (TWorld)
  └── FChunks: список "ключ чанка" → TChunk
        ключ = "x:y:z" (координаты чанка)
        чанк есть только если он уже создан (SetBlock / EnsureChunk)

Для каждой пары "ключ" → TChunk:
  TChunk = массив [0..15, 0..15, 0..15] of TBlock
  «пустые» места = блок с Id = 0 (воздух)
```

- **Чанк полностью заполнен (нет воздуха)** — просто во всех ячейках не 0; структура та же.
- **Только часть блоков, остальное воздух** — в части ячеек Id = 0, массив тот же 16×16×16.
- **Чанк «недозаполнен» в смысле «его ещё нет»** — такого чанка нет в `FChunks`, при `GetBlock` возвращается воздух без создания чанка.

## 4. Краткая сводка

| Вопрос | Ответ |
|--------|--------|
| Чанк в памяти «половинчатый» или только часть блоков? | Нет. Чанк всегда полный 16×16×16; «пустые» места — это блок с Id = 0. |
| Если чанк «полностью заполнен» блоками? | Хранится так же: один TChunk, все ячейки с каким‑то Id. |
| Как управляются чанки? | Только те чанки, к которым был доступ (SetBlock/EnsureChunk), попадают в список и хранятся; остальные не создаются, GetBlock по ним даёт воздух. |
